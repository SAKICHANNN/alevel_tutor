## §1 KaTeX Parse Error 排查与修复

### 症状
Gradio Chatbot 显示 "KaTeX parse error" 而非公式渲染。

### 根因分析
三层问题：

| # | 问题 | 示例 | KaTeX 行为 |
|---|------|------|-----------|
| 1 | LLM 用 `$` 当货币符号 | `价格 $10` | 把 `$10` 当数学块起始，无配对 `$` → 报错 |
| 2 | `&` 在 `$...$` 内部 | `$A & B$` | KaTeX 将 `&` 视为对齐符，不允许在行内公式中使用 |
| 3 | `\ce{}` 命令 | `$\ce{H2O}$` | KaTeX 不支持 mhchem 扩展 |

### 修复：双层防线

**① 系统提示层** (`prompts.py`)：
- `## LaTeX 数学公式规范` — 明确禁止 4 项：
  - `$` 不能当货币（写"10元"代替 "$10"）
  - `&` 不能在 `$...$` 中（KaTeX 对齐符）
  - 不能用 `\ce{}`（KaTeX 不支持）
  - 所有 `$` 必须配对
- 提供正反面示例

**② 后处理层** (`core.py:_fix_katex`)：
- 逐字符扫描器（非正则），正确处理 `$$a = $b$ + c$$` 嵌套
- 预处理：`$10` → `10元`（匹配 `$` + 数字，但不匹配 `$10$`）
- LaTeX 块内修复：`&` → `\text{ and }`、`\ce{...}` → `\text{...}`
- LaTeX 块外：游离 `$` → `＄`（全角字符，不作分隔符）
- 修复计数通过 `_last_katex_fixes` 暴露给调试日志

**③ 测试覆盖**：
- 9 项测试全部通过：货币 `$`、行内公式、独立公式、混合、游离 `$`、嵌套、`&` 修复、`$10$` 保留、千分位

## §2 GUI 调试日志系统

### 动机
KaTeX/SVG 渲染错误发生在浏览器端，服务端无记录。每次调试需猜测 LLM 原始输出。

### 实现 (`web/app.py`)

**日志文件**：`data/gui_debug.log`（JSON lines，保留最近 500 条）

**记录字段**：

```python
_log_debug({
    "session": session_id[:8],     # 会话标识
    "subject": "9708",              # 科目
    "model": "deepseek-v3",         # LLM 模型
    "conv_len": 12,                 # 对话轮数
    "user_msg": message[:300],      # 用户输入截断
    "response_len": len(response),  # 响应总长度
    "svg_count": 2,                 # 图表数量
    "raw_len": len(_last_raw_output), # 原始输出长度
    "katex_fixes": {...},           # KaTeX 自动修复统计
    "tool_calls": [...],            # 工具调用记录
    "elapsed_s": 4.32,              # 响应耗时
    "error": None,                  # 异常信息
    "response": response[:5000],    # 前 5000 字响应
})
```

**数据来源** (`core.py` 新增全局追踪变量)：
- `_last_raw_output` — LLM 原始输出（sanitize 之前）
- `_last_tool_calls` — 每轮 tool call 的 tool、args、result_len
- `_last_response_time` — chat() 执行耗时
- `_last_model` — 当前使用的模型名
- `_last_svg_count` — 响应中 SVG 数量
- `_last_conv_len` — 对话历史长度
