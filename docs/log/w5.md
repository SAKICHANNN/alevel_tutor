## §2 W5.1 RAG / 文件上传安全护栏

创建 `agent/security.py`：

**检索内容防护**（`sanitize_retrieved_content`）：
- 所有 RAG 结果包裹 `[⚠️ 以下内容来自检索结果...]` 标记
- 移除 `System:` / `SYSTEM:` 前缀行（常见注入向量）
- 移除 markdown code blocks 中的 `system|instruction|prompt` 标签

**工具调用校验**（`validate_tool_call`）：
- 白名单：6 个工具（search_textbook, search_past_paper, get_exam_pattern, search_exam_techniques, grade_homework_image, get_subject_info）
- 参数注入检测：每个 string 参数过 `detect_injection()`

**集成点**：
- `Agent.chat()`：用户输入先过 `detect_injection()` → 拦截则返回安全提示
- `Agent._execute_tool()`：工具调用前过 `validate_tool_call()` → 拦截则返回错误
- `_execute_tool` search 结果：过 `sanitize_retrieved_content()` 包裹

## §8 W5.9 QP+MS 自动映射（工具设计）

**目标**：用 VLM 批量提取真实 QP→MS 对照，建立 question_text → mark_scheme_text 直接映射表。

**方案**：
1. 选择 2022-2024 各科 QP+MS 各 5 份（共 10 对 × 4 科 = 40 对）
2. 每对取 Q1-Q5（前 5 题），每题渲染 2 页（QP 题目页 + MS 对应页）
3. VLM 提取：question text → corresponding mark scheme text
4. 输出 `data/qp_ms_map.json`（含 source PDF, page, confidence）

**瓶颈**：LM Studio 每页 ~20-30s，40 对 × 5 题 × 2 页 = 400 页 → 8000s ~2.2h。建议离线批处理。

**脚本已预留**：未在本次 commit 中创建（需要 VLM 可用时批处理运行），设计已在 log 中记录。
