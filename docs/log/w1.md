## 三、OCR 引擎适配（PaddleOCR 3.x）

### 3.1 API Breaking Changes 处理
PaddleOCR 3.5.2 + paddlex 3.5.2 相比之前版本有重大变化：

| 旧 API | 新 API | 变化 |
|--------|--------|------|
| `PaddleOCR(show_log=False, use_angle_cls=True)` | `PaddleOCR(lang='en')` | 参数名废弃 |
| `from paddleocr import PPStructure` | `from paddleocr import PPStructureV3` | 类名变更 |
| `PPStructure(table=True, layout=False)` | `PPStructureV3()` | 参数简化 |
| — | `from paddleocr import FormulaRecognition` | 新增公式模块 |
| `ocr.ocr(img.read(), cls=True)` | `ocr.ocr(np_array)` | 接受 numpy array |
| `fr(arr)` | `fr.predict(arr)` | 方法名变更 |
| Surya `TexifyPredictor` | PP-FormulaNet-S `FormulaRecognition` | 引擎替换 |

### 3.2 修复的文件
- `agent/ocr/pipeline.py`：替换 Surya → PP-FormulaNet，修正 PPStructureV3 import，适配 predict() API
- 修正 DECIMER import：尝试 `DECIMER` 和 `decimer` 两种大小写
- `agent/config.py`：添加 `.env` 自动加载

### 3.3 PP-FormulaNet-S 实测
```python
fr = FormulaRecognition(model_name='PP-FormulaNet-S')
arr = np.array(cropped_formula_image)
result = fr.predict(arr)
# → [{'rec_formula': 'f^{\\prime}(x)=2x^{-{\\frac{1}{3}}}-x^{{\\frac{1}{3}}}'}]
# 正确提取了 2022 年真实 9709 P1 试卷公式
# 延迟：0.48s/公式
```

### 3.4 已知限制
- `rec_formula` 偶有 OCR 噪声（如 "bf" 字符混入），但核心公式正确
- PyMuPDF 文本层提取有 PDF 编码损坏（`f(x)` → `f\u0000x\u0001`）
- PP-FormulaNet-S 是轻量版（221MB），PP-FormulaNet_plus-L（698MB）未测试——可能需更多 GPU 内存但中文公式精度更高

## 四、教学引擎验证

### 4.1 Hint-First 流程
```
用户：How do I find the area bounded by y = x^2 and y = 4?
→ Agent 调用 search_past_paper + get_exam_pattern
→ 🎯 比喻：「把水（抛物线）倒进方杯子（y=4 线），重叠部分就是面积」
→ 📖 分步：找交点→确定上下→积分→计算
→ 完整解题，每步有 LaTeX 公式
```

### 4.2 批改能力测试
```
Q: Differentiate y = 3x^2 + 2x
正确答案 "dy/dx = 6x + 2"  → Agent 正确判定满分
符号错误 "dy/dx = 6x - 2"    → Agent 识别符号错误，标记代数错误
无步骤 "6x + 2"              → Agent 指出缺少 working，method 评 0 分
```

### 4.3 真实试卷辅导
```python
# 提取 2022 P1 Question 1 文本
question = "A curve with equation y = f(x) such that f'(x) = 2x^{-1/3} - x^{1/3}. f(8) = 5. Find f(x)."
agent.chat(question)
# → 🎯 比喻：「像拼图游戏，速度表（导数）→ 总路程（原函数），起点 f(8)=5 定唯一答案」
# → 📖 分步积分 + 代入求常数
```
