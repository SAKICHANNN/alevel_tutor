## 三、W3 核心开发

### 3.1 本地 LM Studio Qwen3-VL-8B 集成（commit `7497a1b`）

你提供了本地 LM Studio 运行的 Qwen3-VL-8B（`qwen/qwen3-vl-8b`），接入方式：

**配置**：
```python
# agent/config.py
LMSTUDIO_BASE_URL = "http://127.0.0.1:1234/v1"
LMSTUDIO_VLM_MODEL = "qwen/qwen3-vl-8b"

# vision_local 优先于 vision（Zhipu）和 vision_qwen（阿里云）
MODELS["vision_local"] = LLMConfig(provider="lmstudio", ...)
get_active_vision_model() → 优先 vision_local
```

**效果**：
- 物理 P4 试卷：正确识别引力场图、题目结构、公式
- 化学 P4 试卷：正确识别催化机理、Ksp 计算
- 数学 P1 试卷：完美提取 LaTeX 公式 `f'(x) = 2x^{-1/3} - x^{1/3}`
- **0 API 成本**，M5 32GB 本地推理

### 3.2 OCR 架构简化（commit `2b26b6f`）

你说「直接把有难度的工作都交给 Qwen，普通 OCR 只做最简单的工作」。

**之前**（复杂且不可靠）：
```
PP-FormulaNet → _is_garbled_formula() → Qwen VLM → MathPix
PaddleOCR → PPStructureV3 → DECIMER → VisionAnalyzer
```

**现在**（简单直接）：
```
PaddleOCR (0.5s) → 纯文字提取（搜索、索引用）
Qwen VLM (20-60s) → 全部结构化内容（公式、表格、图表、化学结构）
```

**移除的组件**：PP-FormulaNet 乱码检测、PPStructureV3 表格、DECIMER 化学 OCR 路由、预处理阈值化（会把图片变全黑）

### 3.3 PaddleOCR 3.x API 兼容（commit `416b9be`）

| 旧 API (2.x) | 新 API (3.5.2) |
|-------------|----------------|
| `ocr.ocr(arr)` | `ocr.predict(arr)` → OCRResult 对象 |
| `PPStructure(arr)` | `PPStructureV3.predict(arr)` |
| `FormulaRecognition().__call__()` | `.predict()` |
| OCRResult 用属性访问 | 用 dict 访问（`r['rec_texts']`） |

### 3.4 预处理 Bug 修复

`ImagePreprocessor.preprocess(is_document=True)` 对数字试卷 PDF 做了阈值二值化，结果全黑图。修复：
- PaddleOCR：`is_document=False`（轻 resize）
- Qwen VLM：传原始 `image_data`，不做预处理
