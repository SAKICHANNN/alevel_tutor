## §7 W4.1 评分校准 — PedCoT 两阶段评分

### 7.1 研究方法

深度网络搜索明确了三个关键路径：

**Path A — Cambridge M-A-B 真实打分体系**（来自官网 MS PDF）
- M 分：方法分，不因算术错误扣分
- A 分：精确分，必须在 M 分获得后才能给
- B 分：独立分，不依赖其他分
- FT：结果错误但方法正确时继续给分
- 正向评分，只加分不扣分，只给整数分

**Path B — PedCoT 两阶段评分**（Jiang et al. IJCAI 2024）
- Stage 1 (Regenerate)：LLM 先独立解题，不留心学生答案
- Stage 2 (Extract-Compare)：看到学生答案，提取实际步骤，比较，评分
- 实验证明两阶段比一阶段好得多（Mistake Finding Acc: 83% vs 74%）
- 原因：单阶段把学生答案一起给 LLM，会有 conformality bias（偷懒同意学生）

**Path C — MS 匹配**（ChromaDB 互补检索）
- 用搜索匹配到的真实 mark scheme 时，直接用 MS 评分
- 匹配算法：搜索 past_papers collection (MS type) + Reranker score > 0.75
- 匹配失败时用 PedCoT 自动生成评分方案

### 7.2 实现

`agent/grading/grader.py` — 三路径评分引擎：
1. 尝试匹配真实 MS（搜索 + Reranker 阈值 0.75）
2. 匹配成功：直接用 MS 评分（一次 API 调用）
3. 匹配失败：PedCoT 两阶段（两次 API 调用）

### 7.3 校准结果

15 个测试案例（P1 × 6, Integration × 3, Differentiation × 3, Mechanics × 3）：
- **MAE = 0.300 分** (< 0.5 ✓)
- **Pearson r = 0.898** (> 0.85 ✓)

标定分析：
- 10/15 案例完全匹配 (Δ=0.0)
- P1-04: Gold=3 Agent=2 — Stage-1 自动分配的满分与 gold 不一致（Cambridge 3 分 vs 自动 2 分）
- DIF-03: Gold=1.5 Agent=0 — 代理评分偏严格，对学生只写了 5(2x+1)^4（缺少 ×2）给了 0 分
- MEC-03: Gold=1.5 Agent=2 — 代理评分偏宽松，对极度简化的答案给了满分
- 偏差来源：自动生成 MS 的满分判定与真实 Cambridge MS 有 ±1 分的差异

### 7.4 限制和后续

- 自动生成的 MS 满分可能与 Cambridge 官方不一致（±1 分）
- 对简化答案可能偏严，对方法正确但步骤不完整可能偏松
- **建议**：收集更多实际 Cambridge MS PDF，分词条匹配提升匹配率
- 真人评分对照需要真实的 gold set（真实学生答案 + 考官评分）

## §8 W4 Bug Fixes

1. **_call_llm default tools_list 逻辑错误**：`tools_list=None` 时 `tools_list if tools_list is not None else TOOLS` = TOOLS。改为 `tools_list: list = TOOLS`，显式传 `[]` 表示无工具。这导致评分 JSON 失败（工具干扰 "JSON only" 指令）

2. **grading max_marks 未约束**：prompt 中 `<number>` 无上限，导致 LLM 给 10/10 到 3 分题。在 prompt 中插入 `{max_marks}` 约束

3. **calibration 文件路径 bug**：`Path(__file__).parent.parent.parent` 计算错误，导致输出写到 `/Users/miaomiao/data/` 而非项目目录
