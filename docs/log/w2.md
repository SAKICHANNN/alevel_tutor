## 三、关键 Bug 修复：真题索引不均衡

### 3.1 问题

旧代码：`max_papers_per_subject=20`，遍历目录时按字母序（ci → er → gt → ms → qp），ms 先到 20 上限，qp/er 全部被跳过。

**结果**：2,543 个 chunks **全是 ms**。学生搜"这道题怎么做"找不到题目原文，只能搜到答案。

### 3.2 修复

改为 `max_per_type=10`，每类独立计数：

```python
type_counts = {}  # {type_name: count}
# 每类各自限 10，不跨类型累计
if type_counts[type_name] >= max_per_type:
    continue
```

修复后：qp:10 + ms:10 + er:10 + gt:10 = 40 份/科，各类型均匀。

### 3.3 ChromaDB 多条件过滤

`where={"subject": "9709", "type": "qp"}` 在 ChromaDB 中不合法——需要用 `$and`：

```python
where_filter = {"$and": [{"subject": "9709"}, {"type": "qp"}]}
```
