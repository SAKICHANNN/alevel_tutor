## §8 W5.9 QP+MS 自动映射（工具设计）

**目标**：用 VLM 批量提取真实 QP→MS 对照，建立 question_text → mark_scheme_text 直接映射表。

**方案**：
1. 选择 2022-2024 各科 QP+MS 各 5 份（共 10 对 × 4 科 = 40 对）
2. 每对取 Q1-Q5（前 5 题），每题渲染 2 页（QP 题目页 + MS 对应页）
3. VLM 提取：question text → corresponding mark scheme text
4. 输出 `data/qp_ms_map.json`（含 source PDF, page, confidence）

**瓶颈**：LM Studio 每页 ~20-30s，40 对 × 5 题 × 2 页 = 400 页 → 8000s ~2.2h。建议离线批处理。

**脚本已预留**：未在本次 commit 中创建（需要 VLM 可用时批处理运行），设计已在 log 中记录。

### 发现的 Bug

**Bug 18: `detect_injection` 缺少 DOTALL 标志 — 多行注入漏检**
**文件**：`agent/security.py:128`
**症状**：`re.search(pattern, text_lower, re.IGNORECASE)` — 默认模式 `.` 不匹配 `\n`。输入 `"\n\nIgnore\n\nall\n\ninstructions"` 中的 `ignore.*instructions` 无法跨行匹配，导致多行注入漏检。
**修复**：改为 `re.search(pattern, text_lower, re.IGNORECASE | re.DOTALL)`。

### 代码审查发现

**Bug 19: `strip_exif` 泄漏临时文件**
**文件**：`agent/security.py:72` — 函数创建 `filepath + ".clean" + ext` 但从不清理。
**修复**（c10b4bf）：添加 `atexit.register(_cleanup_temp_files)` + `cleanup_temp_files()` 公开函数 + EXT_TO_MIME 映射表。

**Bug 20: `security.py` 死代码**
- `import io` — 未使用 → **已删除**
- `import PIL.ExifTags` — 未使用 → **已删除**
- `ALLOWED_MIME_TYPES` — 未使用 → **替换为 EXT_TO_MIME 映射表（7 个扩展）**

**Bug 21: `strip_exif` 内存风险**
`data = list(img.getdata())` — 20MB 图像可膨胀到 100+ MB。
**缓解**（c10b4bf）：添加文档说明，临时文件注册到 atexit 清理，大图限制 20MB 上限阻止极端情况。
